# 📦 Images & Containers

## 📌 What is a Docker Image?

A Docker Image is a **read-only template** used to create containers.

👉 Example:

* mysql:8.0
* nginx
* openjdk:21

---

## 📌 What is a Container?

A Container is a **running instance of an image**.

👉 When you run an image, it becomes a container.

---

## 🔥 Image vs Container

| Image                | Container           |
| -------------------- | ------------------- |
| Blueprint            | Running instance    |
| Static               | Dynamic             |
| Stored in Docker Hub | Runs on your system |

---

## ⚙️ Lifecycle of a Container

1. Pull image
2. Run container
3. Stop container
4. Restart container
5. Remove container

---

## 💡 Key Understanding

* One image → multiple containers
* Containers are isolated environments
* Containers can be started/stopped anytime

---
